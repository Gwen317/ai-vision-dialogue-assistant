# AI 视觉对话助手：系统核心设计与创新特性数据规程规范 (System Data Schema Design)

为了保障系统在研发实施过程中的数据持久化完整性、端云长连接协议解析的精确性，本设计规范详细定义了系统总体设计蓝图中**第二大点（核心模块与创新设计规范）**所列出的全部 6 大核心模块与创新特征的**数据字典（Data Fields）**、**数据库表结构（DB Schemas）**及**网络传输载荷（API Payloads）**。

---

## 核心设计一：双向全双工通信与记忆分叉预防 (Memory Truncator & Duplex Control)

### 1. 关系型数据库设计：会话消息日志表 (`session_messages`)
为了处理用户打断时的“记忆分叉”，数据库需要完整记录 AI 生成的原内容与因打断产生的截断内容，保障历史会话的语义连贯性。

* **建表 SQL (PostgreSQL/SQLite 规范)**：
```sql
CREATE TABLE session_messages (
    message_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    session_id VARCHAR(64) NOT NULL,
    role VARCHAR(10) NOT NULL CHECK (role IN ('user', 'model')),
    content TEXT NOT NULL,                     -- 最终参与下一次上下文生成的文本 (若打断，则为截断后文本)
    is_interrupted BOOLEAN NOT NULL DEFAULT FALSE, -- 标识该消息是否被中途打断
    original_content TEXT,                     -- 存储云端大模型原始生成的完整文本 (用于审计与分析)
    truncated_at_char INT DEFAULT NULL,        -- 发生打断时的字符位置偏移量
    timestamp TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_msg_session_id ON session_messages(session_id);
CREATE INDEX idx_msg_timestamp ON session_messages(timestamp ASC);
```

### 2. WebSocket 传输控制帧格式
* **用户主动打断控制帧 (Client -> Gateway JSON)**：
  ```json
  {
    "type": "interrupt",
    "timestamp": 1718182048500, // 毫秒时间戳
    "offset": 45                // 打断时刻前端播放器已朗读的字数 (字符偏移量)
  }
  ```

## 核心设计二：端侧边缘预处理与主动质量引导 (Edge Preprocessing & Q-Gate)

### 1. 端侧配置状态数据结构 (`EdgeConfigState`)
前端状态机用于控制边缘端过滤阈值和 VAD 判定界限的 TypeScript 接口定义：
```typescript
interface EdgeConfigState {
  // 语音端 (VAD) 过滤参数
  vadEnergyThreshold: number;       // 音频振幅能量门限 [0.0 - 100.0] (默认 15.0)
  minSpeechDurationMs: number;      // 判定为说话的最短语音长度 (ms) (默认 150ms)
  silenceBufferFrames: number;      // 判定为说话结束的静音缓冲帧数 (每帧约20ms，默认 60帧即1.2s)
  
  // 视觉端过滤参数
  pixelDiffRateThreshold: number;   // 前后帧差异像素比率阈值 [0.0 - 1.0] (低于该值判定为画面静止，默认 0.05)
  laplacianBlurThreshold: number;   // 拉普拉斯算子方差阈值 (低于该值判定为模糊，默认 12.0)
  luminanceMinThreshold: number;    // 最低可接受灰度亮度值 [0 - 255] (过暗阈值，默认 40)
  luminanceMaxThreshold: number;    // 最高可接受灰度亮度值 [0 - 255] (过曝阈值，默认 240)
}
```

### 2. 图像质量本地判定与反馈传输帧 (Client -> Gateway JSON)
当图像帧通过或未通过端侧预检时，客户端向网关上传检测指标以进行性能监控：
```json
{
  "type": "quality_feedback",
  "timestamp": 1718182045000,
  "blur_variance": 8.45,            // 实际计算出的拉普拉斯方差
  "average_luminance": 32.5,        // 实际计算出的平均亮度 (Y通道分量)
  "motion_pixel_diff_rate": 0.02,   // 前后帧变动像素比例
  "is_sent_to_cloud": false,        // 是否由于质量不达标/画面静止被端侧拦截
  "filter_status": "BLOCKED_DARK"   // 拦截状态: PASSED | BLOCKED_BLUR | BLOCKED_DARK | BLOCKED_STATIC
}
```

## 核心设计三：多模态长程情景记忆系统 (Multimodal Episodic Memory)

### 1. 关系型数据库设计：情景记忆日志表 (`episodic_memory_log`)

* **建表 SQL (PostgreSQL 规范)**：
```sql
CREATE TABLE episodic_memory_log (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    session_id VARCHAR(64) NOT NULL,
    timestamp TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT CURRENT_TIMESTAMP,
    user_speech TEXT NOT NULL,
    ai_response TEXT NOT NULL,
    image_path VARCHAR(255),                  -- 端侧捕获图片文件存储的相对/绝对路径
    image_description VARCHAR(500),           -- Gemini Flash 自动生成的1句摘要
    entity_tags VARCHAR(64)[] NOT NULL,       -- 提取的实体标签数组，如 ["LED", "电阻"]
    created_at TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_mem_session_id ON episodic_memory_log(session_id);
CREATE INDEX idx_mem_timestamp ON episodic_memory_log(timestamp DESC);
```

### 2. 向量数据库集合设计 (Qdrant Schema)
* **集合名称**：`multimodal_memory_collection`
* **向量规范**：
  * `vector_text` (主文本语义向量)：768 维 (对应 `text-embedding-004` 模型)
  * `vector_image` (多模态视觉向量)：512 维 (对应 `multimodal-embedding-001` 模型)
* **Payload 荷载字段设定**：
  ```json
  {
    "memory_id": "UUID (对应关系表 episodic_memory_log.id)",
    "timestamp": "ISO8601 字符串",
    "description": "string (物品一句话简述)",
    "tags": ["string"]
  }
  ```

## 核心设计四：听觉混响卷积与噪音自适应 (Acoustic Matching & Lombard Effect)

### 1. 关系型数据库设计：声学空间预设表 (`acoustic_presets`)

* **建表 SQL (PostgreSQL/SQLite)**：
```sql
CREATE TABLE acoustic_presets (
    preset_id VARCHAR(32) PRIMARY KEY,        -- 唯一ID，如 'studio', 'living_room', 'cathedral'
    name_cn VARCHAR(64) NOT NULL,             -- 中文显示名称
    reverb_duration REAL NOT NULL,            -- 混响时间 RT60 (秒) [0.0 - 5.0]
    decay_ratio REAL NOT NULL,                -- 衰减率参数 [0.1 - 10.0]
    high_freq_damping REAL NOT NULL,          -- 高频衰减率，用于模拟空气对高频声波的吸收
    wet_gain REAL NOT NULL,                   -- 湿音增益 (混响声比例) [0.0 - 1.0]
    dry_gain REAL NOT NULL                    -- 干音增益 (直达声比例) [0.0 - 1.0]
);
```

### 2. Lombard 效应自适应状态数据结构 (Client-Side TS State)
```typescript
interface LombardState {
  backgroundNoiseDb: number;        // 实时检测到的噪音声压 (dB) [-100.0 - 0.0]
  isLombardActive: boolean;         // 是否触发自适应调整阈值 (阈值设为 -50dB)
  dynamicOutputGain: number;        // 实时输出增益乘数 [1.0 - 2.5]
  highShelfGainDb: number;          // 高频滤波器增益 (dB) [0.0 - 12.0]
  cutoffFrequency: number;          // 高通截止频率，动态过滤低频环境杂音 (Hz) [100 - 300]
}
```

## 核心设计五：D3 物理环境实体图谱 (Entity Graph)

### 1. 关系型数据库设计：实体表 (`graph_nodes`) 与 关系表 (`graph_links`)

* **实体节点表 (PostgreSQL)**：
```sql
CREATE TABLE graph_nodes (
    node_id VARCHAR(64) PRIMARY KEY,          -- 唯一标识，如 'resistor_r1'
    label VARCHAR(128) NOT NULL,              -- 显示名称，如 '电阻 R1'
    category VARCHAR(32) NOT NULL,            -- 类型：'board'|'component'|'tool'|'hardware'
    associated_memory_id UUID REFERENCES episodic_memory_log(id) ON DELETE SET NULL, -- 首次发现该实体的情景记忆卡片ID
    first_seen TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);
```

* **实体关系连线表 (PostgreSQL)**：
```sql
CREATE TABLE graph_links (
    link_id VARCHAR(64) PRIMARY KEY,
    source_node_id VARCHAR(64) NOT NULL REFERENCES graph_nodes(node_id) ON DELETE CASCADE,
    target_node_id VARCHAR(64) NOT NULL REFERENCES graph_nodes(node_id) ON DELETE CASCADE,
    relation_type VARCHAR(64) NOT NULL,       -- 关系类型，如 'connected_to'
    confidence REAL DEFAULT 1.0,               -- 置信度系数 [0.0 - 1.0]
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT unique_relation UNIQUE(source_node_id, target_node_id, relation_type)
);
```

### 2. D3.js 图谱网络载荷格式 (JSON Schema)
