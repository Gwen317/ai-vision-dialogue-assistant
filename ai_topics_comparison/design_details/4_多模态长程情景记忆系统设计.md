# AI 视觉对话助手：多模态长程情景记忆 (Multimodal Episodic Memory) 创新设计

您提出的这个创新点非常具有前沿性！我们将其定义为：**基于多模态 RAG 的长程情景记忆与实体关联系统 (Multimodal Episodic Memory & Entity Association)**。

这个设计能够让 AI 摆脱“短暂会话记忆”的局限，让它具备**类似人类的视觉与语言长期记忆能力**。它能够记住用户在“过去几分钟、几天甚至更久”展示过的物体和说过的话，并在当下进行跨时空的对比与关联。

---

## 1. 创新点核心机制

本系统融合了**图像特征向量检索 (CLIP)** 与**文本向量检索 (Text RAG)**，在后端构建了一个**“多模态事件日志数据库” (Multimodal Event Log DB)**。

每当用户展示一个物品并与之对话时，系统会在后台静默生成一个 **“情景记忆卡片” (Episodic Memory Card)**，结构如下：
* `id`：唯一标识符
* `timestamp`：精确时间戳（用于计算“过去多久”）
* `image_vector`：使用图像嵌入模型（如 CLIP）提取的物体视觉特征向量
* `image_thumbnail`：端侧轻量压缩保存的缩略图（用于再次显示或对比）
* `entity_labels`：视觉大模型识别出的物品标签（如 `["绿色电容", "C12", "主板"]`）
* `transcript`：当时用户说的话与 AI 的回答
* `text_vector`：对话文本的语义向量

## 2. 核心用户故事体验 (User Stories)

### 故事一：跨时空物品对比 (Cross-Temporal Comparison)
* **场景**：用户拿起一个红色的手机壳，说：*“这个手机壳和我前天给你看的那个蓝色的有什么区别？”*
* **AI 表现**：AI 检索前天的视觉记忆卡片，提取蓝色手机壳的图像特征和信息，回答：*“前天下午 3 点你给我展示的是一个带有卡槽的蓝色硅胶手机壳。而你现在拿的这个红色手机壳是磨砂材质的，并且没有卡槽，但它的边缘防摔角做得更厚。”*

### 故事二：物理世界物品寻回 (Physical Object Finder)
* **场景**：用户问：*“我刚才把车钥匙丢哪儿了？你看到了吗？”*
* **AI 表现**：AI 检索半小时内的视频关键帧与 VAD 对话记录，发现其中一帧背景里用户将钥匙丢在了茶几上，回答：*“在 15 分钟前，当你向我展示你的新水杯时，我注意到你顺手把车钥匙放在了你身后的白色茶几上（附带当时画面截图）。”*

### 故事三：设备故障排查状态延续 (State Continuity)
* **场景**：用户重新打开设备，展示一块主板，说：*“我们继续修这个板子吧。”*
* **AI 表现**：自动识别主板特征，关联出昨天的维修记录，回答：*“好的，昨天我们检查到了 R15 电阻阻值异常，并计划今天对其进行更换。请问你已经准备好烙铁了吗？”*

## 4. 关键技术选型与难点攻克

### A. 双路向量检索 (Bi-path Vector Retrieval)
在后端实现时，不能仅依赖文字检索，也不能仅依赖图片检索。
* **视觉路径**：将实时截取的视频帧通过 `CLIP` 模型（可以在 Java 中使用 PyTorch 引擎加载，或调用云端 Embedding 服务）提取一个 512 维向量。与历史库进行余弦相似度（Cosine Similarity）匹配，筛选出视觉最相似的物品。
* **文本路径**：用户说 *“之前那个”* 时，利用文本 Embedding 检索历史对话中包含的实体和操作。
* **融合排序**：将视觉相似度分值与文本相似度分值加权融合，精准锁定目标历史卡片。

### B. 记忆主动唤醒与上下文重构
如果历史记忆图片直接重新塞入当前 prompt，会导致 Token 浪费。
* **重构策略**：仅在检索相似度超过高阈值（例如 `0.85`）时，网关才将该历史帧的“缩略图”与“当时 AI 生成的文字描述”塞入 LLM 的 `System Prompt` 中。这样能够最大化降低日常 Token 损耗。
